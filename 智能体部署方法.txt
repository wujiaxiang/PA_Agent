================================================================
  PA Agent（二次开发版）智能体部署方法
  适用环境：macOS / Linux / Windows 任意系统 + Python 3.11+
  运行形态：Web 前端（FastAPI + SSE），跨平台，与操作系统无关
================================================================

【说明】
本文档供自动化 Agent 按步骤执行部署。本仓库为 fork 自
rosemarycox5334-debug/PA_Agent 的二次开发版本，主入口是 Web 后端，
不再依赖 PyQt6 桌面 GUI 和 MT5 终端，可跨平台部署。

【二次开发版的核心变化】
1. 主入口改为 Web 后端（web/server.py，FastAPI + SSE）
2. 配置通过 .env 环境变量管理（三层覆盖优先级），无需前端录入
3. 启动时自动健康检查（模型 API + TradingView 账号连通性）
4. 新增 Mock 数据源：云端沙箱无外网访问时也能跑通流程
5. 桌面 GUI（pa_agent/gui/）仍保留，但不是主推形态，可选

【重要：项目路径】
本文档中 PROJECT_ROOT 代表 PA_Agent 项目的根目录。
执行前必须先确定项目实际路径，后续所有命令中的 PROJECT_ROOT 都替换为实际路径。
例如：~/Projects/PA_Agent 或 /workspace

确定方法：找到项目中存在 web/server.py 和 pyproject.toml 的那个目录，
就是 PROJECT_ROOT。

================================================================
  第一步：确认前置条件
================================================================

1.1 确认 Python 版本

  python3 --version

预期输出：Python 3.11.x 或更高版本（3.12、3.13 均可）

若输出"command not found: python3"：
  方案 A — 官方安装包（推荐）：
    访问 https://www.python.org/downloads/ 下载 Python 3.11+ 安装包
  方案 B — Homebrew（macOS）：
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    brew install python@3.13

1.2 确认 git 已安装

  git --version

预期输出：git version 2.x

================================================================
  第二步：升级 pip
================================================================

  python3 -m pip install --upgrade pip -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

验收：
  python3 -c "import pip; print(pip.__version__)"
预期输出：pip 版本号 24.x 或更高

【故障排查】
- 若 pip 升级网络超时，换阿里云源：
    python3 -m pip install --upgrade pip -i https://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com

================================================================
  第三步：安装项目依赖
================================================================

  cd PROJECT_ROOT
  pip3 install -e . -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

说明：
- -e . 表示以开发模式安装，代码改动即时生效
- -i 指定清华源，避免默认源网络问题
- macOS/Linux 下 pip 会自动跳过 pyproject.toml 中标记 sys_platform == 'win32'
  的包（MetaTrader5、pywin32），这些是桌面 GUI + MT5 直连场景才需要的
- Web 后端核心依赖：fastapi, uvicorn, openai, pydantic, pandas, numpy,
  jsonschema, cryptography, akshare, baostock, curl_cffi

  注意：openai 包偶尔不会被 pyproject.toml 自动装上，
  若启动时报 "RuntimeError: openai package is not installed"，单独装：
    pip3 install openai

  注意：tvDatafeed 是 TradingView 数据源的核心依赖，从 GitHub 安装
  （git+https），网络环境需能访问 GitHub。Web 后端默认用 Mock 数据源
  时不需要它，但若要用 TradingView 必须装。

验收：检查 Web 后端核心包是否可导入

  python3 -c "import fastapi; print('fastapi OK')"
  python3 -c "import uvicorn; print('uvicorn OK')"
  python3 -c "import openai; print('openai OK')"
  python3 -c "import pydantic; print('pydantic OK')"
  python3 -c "import pandas; print('pandas OK')"
  python3 -c "import numpy; print('numpy OK')"
  python3 -c "import jsonschema; print('jsonschema OK')"
  python3 -c "import cryptography; print('cryptography OK')"

预期输出：每行均显示 "xxx OK"

【故障排查】

问题 1：pip install 网络超时 / ConnectionError
解决：换阿里云源
  pip3 install -e . -i https://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com

问题 2：tvDatafeed 安装失败（git clone 超时 / 网络不可达）
说明：tvDatafeed 从 GitHub 安装（git+https://github.com/rongardF/tvdatafeed.git），
  国内网络可能无法直接克隆。
解决：
  方案 A — 配置 git 代理（如果已有代理）：
    git config --global http.proxy http://127.0.0.1:7890
    git config --global https.proxy http://127.0.0.1:7890
    pip3 install git+https://github.com/rongardF/tvdatafeed.git
  方案 B — 手动下载 + 本地安装：
    浏览器打开 https://github.com/rongardF/tvdatafeed
    → Code → Download ZIP → 解压到本地
    → cd 到解压目录 → pip3 install .
  方案 C — 不使用 TradingView 数据源时可跳过：
    在 .env 中设 PA_AGENT_GENERAL_LAST_DATA_SOURCE=mock 即可不依赖 tvDatafeed。

问题 3：tvDatafeed 安装后数据拉取报 "Connection to remote host was lost"
原因：tvDatafeed 2.1.0 已知 bug — WebSocket headers 参数传的是 JSON 字符串而非
  Python dict，导致 websocket-client ≥1.8 版本拒绝握手（HTTP 400）。
解决：找到 tvDatafeed 安装目录下的 main.py，修改一行代码。
  定位文件：
    python3 -c "import tvDatafeed; import os; print(os.path.dirname(tvDatafeed.__file__))"
  编辑该目录下的 main.py，找到第 35 行附近的：
    __ws_headers = json.dumps({"Origin": "https://data.tradingview.com"})
  改为：
    __ws_headers = {"Origin": "https://data.tradingview.com"}

问题 4：云端沙箱无法访问 TradingView / yfinance / akshare
说明：TRAE 远程沙箱出站防火墙可能拦截 tradingview.com:443，
  yfinance 返回 429，akshare 也可能被限流。
解决：Web 后端已内置 Mock 数据源，无需外网即可跑通流程：
  在 .env 中设 PA_AGENT_GENERAL_LAST_DATA_SOURCE=mock
  Mock 数据源会生成带趋势+噪声的模拟 K 线，足以验证前后端流程。

================================================================
  第四步：配置环境变量（.env）
================================================================

4.1 复制模板

  cd PROJECT_ROOT
  cp .env.example .env

4.2 编辑 .env，填入实际值

  关键字段（详细说明见 .env.example）：

  PA_AGENT_PROVIDER_MODEL=tencent/hy3:free
  PA_AGENT_PROVIDER_BASE_URL=https://openrouter.ai/api/v1
  PA_AGENT_PROVIDER_API_KEY=sk-or-v1-xxxxxxxxxxxxxxxxxxxx

  PA_AGENT_GENERAL_LAST_DATA_SOURCE=mock          # 云端调试用 mock，本地用 tradingview
  PA_AGENT_GENERAL_LAST_SYMBOL=XAUUSD
  PA_AGENT_GENERAL_LAST_TIMEFRAME=1h
  PA_AGENT_GENERAL_ANALYSIS_BAR_COUNT=20          # 默认 20，超过 50 会触发 TRAE 网关 SSE 超时

  PA_AGENT_TRADINGVIEW_USERNAME=                  # 仅数据源为 tradingview 时需要
  PA_AGENT_TRADINGVIEW_PASSWORD=

说明：
- 三层覆盖优先级：shell 环境变量 > .env 文件 > config/settings.json
- base_url 必须带 /v1（如 https://openrouter.ai/api/v1），SDK 内部自动拼 /chat/completions
- analysis_bar_count 默认 20，理由：100 根 K 线的 prompt 巨大，分析耗时超过
  TRAE 预览网关 SSE 连接超时，会报 ERR_INCOMPLETE_CHUNKED_ENCODING

验收：启动时健康检查会自动验证模型 API 和 TradingView 账号
  curl http://localhost:8000/api/health/check
预期输出：JSON 包含 "model_api": {"ok": true, ...}

================================================================
  第五步：启动 Web 后端
================================================================

5.1 启动命令

  cd PROJECT_ROOT
  python3 -m uvicorn web.server:app --host 0.0.0.0 --port 8000

5.2 启动日志关键字

启动成功时应看到：
  INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
  ... 健康检查日志：model_api OK / tradingview WARNING（mock 数据源时 TV 失败降级）

5.3 验收清单

  [ ] python3 --version 输出 3.11+
  [ ] 所有 Web 后端核心包导入测试通过（8 个 OK）
  [ ] .env 已配置 PA_AGENT_PROVIDER_API_KEY
  [ ] uvicorn 启动成功，端口 8000 可访问
  [ ] curl http://localhost:8000/api/health 返回 {"status":"ok"}
  [ ] curl http://localhost:8000/api/health/check 返回 model_api ok
  [ ] 浏览器打开 http://localhost:8000 显示前端界面
  [ ] 前端点"开始分析"能收到 SSE 流式事件（约 5 分钟完成 20 根 K 线分析）

全部通过 → 部署成功，可交付用户。

================================================================
  附：可选 — 桌面 GUI 部署（原项目形态，非主推）
================================================================

本仓库保留了原项目的 PyQt6 桌面 GUI（pa_agent/gui/），适合本地直连
MT5 终端的 Windows 用户。Web 用户可跳过本节。

A.1 额外依赖（仅桌面 GUI 需要）

  pip3 install PyQt6 pyqtgraph
  # Windows + MT5 场景额外需要：
  pip3 install MetaTrader5 pywin32

A.2 启动桌面 GUI

  cd PROJECT_ROOT
  python3 -m pa_agent.main

A.3 macOS 桌面 GUI 注意事项

- Mac 无 MT5 原生版本，桌面 GUI 默认用 TradingView 数据源
- 需修改 pa_agent/data/factory.py 把默认数据源从 "mt5" 改为 "tradingview"
- 需修改 pa_agent/config/settings.py 把 last_data_source 默认值改为 "tradingview"
- 生成双击运行脚本：
    cat > "运行智能体.command" << 'EOF'
  #!/bin/bash
  cd "$(dirname "$0")"
  python3 -m pa_agent.main
  EOF
    chmod +x 运行智能体.command

A.4 Windows 桌面 GUI 注意事项

- 需 MT5 终端已安装并登录
- 生成双击运行脚本（GBK 编码）：
    @echo off
    chcp 936 >nul 2>&1
    cd /d "%~dp0"
    python -m pa_agent.main
    pause

================================================================
  附：常见运行时问题
================================================================

Q: 启动报 "RuntimeError: openai package is not installed"
A: pyproject.toml 有声明但环境未装上，单独安装：
   pip3 install openai

Q: 健康检查 model_api 报 404
A: .env 的 base_url 拼接出问题。检查是否已带 /v1，
   如 https://openrouter.ai/api/v1（不要写成 https://openrouter.ai/api）。

Q: 健康检查 model_api 报 400 "output tokens too large"
A: _PRACTICAL_UNLIMITED_MAX_TOKENS 已降到 32768，正常不会触发。
   若仍报错，检查是否用了输出上限更小的模型，换 deepseek-chat 或调小 max_tokens。

Q: 前端报 ERR_INCOMPLETE_CHUNKED_ENCODING
A: SSE 连接被 TRAE 预览网关超时截断。根因是 analysis_bar_count 过大导致
   分析耗时过长。解决：
   1. 确认 .env 的 PA_AGENT_GENERAL_ANALYSIS_BAR_COUNT=20（不要设 100）
   2. 路由已强制 clamp 前端传值到 settings 上限，前端传 100 也会被截到 20
   3. 若仍超时，检查网络是否真的能访问模型 API

Q: TradingView 数据拉取超时 / SSL EOF
A: 云端沙箱防火墙拦截 tradingview.com:443。解决：
   1. 在 .env 中设 PA_AGENT_GENERAL_LAST_DATA_SOURCE=mock 跳过 TV
   2. 本地部署时配置 TradingView 账号登录（PA_AGENT_TRADINGVIEW_USERNAME/PASSWORD）

Q: Mock 数据源返回 None / 'NoneType' object has no attribute 'bars'
A: MockSource.latest_snapshot(n) 必须返回 n+1 根（含 1 根 forming bar），
   否则 build_analysis_frame 丢掉 forming 后不足 n 根 closed 会返回 None。
   当前实现已修复，若自定义数据源请遵守 n+1 约束。

Q: API 调用报网络错误
A: 检查网络连接；确认 API Key 和 Base URL 配置正确；若用代理需确认代理可用。

Q: config/settings.json 损坏
A: 删除该文件，程序重启会自动生成默认配置：
   rm "PROJECT_ROOT/config/settings.json"

Q: 日志在哪里
A: PROJECT_ROOT/logs/pa_agent.log

Q: 如何更新程序
A:
   cd PROJECT_ROOT
   git pull
   pip3 install -e . -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

Q: 如何同步上游原项目的改动
A: 见 README.md「与上游的同步策略」章节。
   git remote add upstream https://github.com/rosemarycox5334-debug/PA_Agent.git
   git fetch upstream
   git merge upstream/main
   冲突时：web/、.env、env_loader.py、startup_health_check.py、mock_source.py
   保留本仓库；pa_agent/ai/、prompt_engineering/、pa_agent/data/base.py
   以上游为准。
