❓ 这个 tab 是什么？怎么用？

实时分析 (Live Analysis) — 流式展示两阶段 AI 分析的完整推理过程，是观看 AI 思考过程的主入口。

两阶段结构：

  • 🔍 阶段一：市场诊断 (Stage 1) — AI 识别当前 K 线的市场结构（方向/周期位置/支撑阻力/闸门检查）
  • 🎯 阶段二：交易决策 (Stage 2) — AI 基于阶段一诊断，输出具体交易方案（订单类型/方向/入场价/止损止盈）

每阶段含 2 个折叠块：

  • 📝 提示词 Prompt — System + User 消息（看 AI 收到了什么）
  • 💡 回答 Answer — 分两个子区：
    • 🧠 推理思考 (Reasoning) — 模型内心独白（thinking 模式可见）
    • 💬 回答输出 (Output) — 模型正式回答（含结构化 JSON）

顶部状态：阶段进度徽章（进行中/✓完成/✗失败/重试中）+ Token 上下文占用条

底部追问：分析完成后可在输入框继续提问，AI 基于本次分析上下文回答

使用建议：分析过程中实时滚动查看推理；分析完成后想看结构化结论去【决策】，想看决策路径去【决策树】

        ❓ 这个 tab 是什么？怎么用？
        

          
决策 (Decision) — 本次分析最终交易决策的结构化报告，采用三段式视觉层次。

          
三段结构（颜色编码）：

          
    
            
  • 🎯 VERDICT 决策结论（蓝色 / 常显）— "做什么"
              
      
                
    • 结论横幅：订单类型 + 方向 + 交易置信度（不下单时显示"观望"）
      • 
                
    • 派生字段：趋势 / 周期 / 阶段（含过渡风险）
      • 
                
    • 价格栅格：入场价 / 止损 / 止盈 TP1·TP2 + 盈亏比（交易员方程）
      • 
                
    • 三条置信度进度条：诊断置信度 / 交易决策置信度 / 预估胜率
      • 
              
    
            
    • 
            
  • 🔬 VITALS 市场状态（青色 / 常显）— "为什么"
              
      
                
    • 2.1 趋势结构：方向 / 周期位置 / 备选周期 / 趋势标签
      • 
                
    • 2.2 市场阶段：market_phase / 过渡风险 / 波动率 / Spike 阶段 / 高潮风险
      • 
                
    • 2.3 关键价位：支撑位（绿）/ 阻力位（红）并排显示
      • 
                
    • 2.4 形态与信号：识别形态 + 关键信号
      • 
              
    
            
    • 
            
  • 📚 EVIDENCE 详细依据（紫色 / 折叠）— "证据链"
              
      
                
    • 7 个独立子折叠区：3.1 决策理由（默认展开）/ 3.2 入场规则 / 3.3 K线分析 / 3.4 趋势上下文 / 3.5 风险评估 / 3.6 关键因素与关注点 / 3.7 置信度说明
      • 
                
    • 顶部"全部展开/全部折叠"主控按钮一键切换
      • 
              
    
            
    • 
          

          
颜色含义：绿色 = 做多 / 支撑 / 通过，红色 = 做空 / 阻力 / 失败，蓝色 = 不下单 / 中性，黄色 = 等待 / 警告

          
使用建议：这是看"AI 给出什么交易方案"的主入口；想看推理过程去【实时分析】，想看决策路径去【决策树】，想看完整 JSON 去【原始】

        ❓ 这个 tab 是什么？怎么用？
        

          
未来 (Future) — AI 对未来走势的概率性预测，含两个时间维度，辅助判断进场后的胜率。

          
两块内容：

          
    
            
  • 📊 下一根 K 线预期 — 下根 K 线是阳线/阴线/中性的概率 + 理由
              
    • "不可预测"标签表示市场处于不确定状态，此时概率分布无意义
    
            
    • 
            
  • 🔄 下一个市场周期预期 — 8 种 Cycle 类型按概率排序，Top-3 蓝色高亮
              
      
                
    • 极速趋势 / 微通道 / 紧通道 / 正常通道 / 宽通道 / 趋势中震荡 / 震荡区间 / 极端震荡
      • 
                
    • 方向标签：看涨 / 看跌 / 中性
      • 
              
    
            
    • 
          

          
程序补全标记：若 AI 未输出但程序根据规则补全，会显示【程序补全】前缀

          
使用建议：用于判断"现在进场后未来 N 根 K 线胜率如何"；市场不可预测时优先看【决策】tab 的置信度

        ❓ 这个 tab 是什么？怎么用？
        

          
决策树 (Decision Tree) — 决策路径的逐步问答回放，看 AI 是如何一步步得出结论的。

          
顶部 banner：终点结果（交易 / 等待 / 放弃 / 继续评估）

          
6 列表格：

          
    
            
  • 步 Step — 序号
    • 
            
  • 阶段 Phase — "一·闸门" = 阶段一闸门检查（筛选是否适合交易），"二·策略" = 阶段二策略决策（具体怎么做）
    • 
            
  • 节点 Node — 决策树节点 ID（如 1.1 / 2.3 / 10.3）
    • 
            
  • 回答 Answer — 绿色=是/通过，红色=否/失败，黄色=中性/等待，灰色=不适用
    • 
            
  • K线依据 Bar Range — 该判断基于哪几根 K 线（如 K50-K1 = 最近 50 根到最近 1 根）
    • 
            
  • 理由 Reason — AI 给出的判断理由
    • 
          

          
使用建议：用于复盘 AI 决策逻辑；阶段一闸门未通过会"短路"（不进入阶段二）；想看完整数据去【原始】tab

        ❓ 这个 tab 是什么？怎么用？
        

          
可视化 (Tree Visualization) — 以流程图形式可视化 AI 决策路径，直观查看决策走向。

          
视觉元素：

          
    
            
  • 实线框 = 实际走过的节点；虚线框 = 未走分支
    • 
            
  • 节点颜色对应回答（绿=通过 / 红=失败 / 黄=等待 / 灰=不适用）
    • 
            
  • 点击「▶ 播放」可沿路径逐步高亮，配合「⏸ 暂停」「⟲ 重置」控制
    • 
          

          
缩放与平移：

          
    
            
  • 点击 ➕/➖ 按钮，或按 Ctrl+滚轮 缩放
    • 
            
  • ⤢ 适配 自动适配窗口
    • 
            
  • 按住鼠标拖拽平移视图
    • 
          

          
使用建议：直观查看决策走向；详细节点数据和理由去【决策树】tab

        ❓ 这个 tab 是什么？怎么用？
        

          
原始 (Raw) — 展示本次分析两阶段的 AI 请求/响应原始数据，是调试与问题排查的主入口。

          
顶部切换：阶段一诊断 / 阶段二决策 / ⚠ 异常（仅失败时显示）

          
每阶段含 4 个折叠区：

          
    
            
  • 📝 System Prompt — 系统提示词（含策略文件内容）
    • 
            
  • 📝 User Prompt — 用户消息（含 K 线数据 / 阶段一诊断结果）
    • 
            
  • 💡 AI 响应 — 模型原始返回，含 reasoning_content 思考过程 + content 正文
    • 
            
  • ⚙️ 验证 + 缓存命中 — stage1/stage2 是否通过校验、缺失/无效字段、KV Cache 命中率
    • 
          

          
顶部操作按钮：

          
    
            
  • 📋 复制调试信息 — 把整条 record JSON 复制到剪贴板（用于 bug report）
    • 
            
  • 💾 导出 JSON — 下载完整 record 为 .json 文件
    • 
          

          
使用建议：排查"AI 给的 JSON 解析失败"或"提示词哪里出问题"时优先看本 tab；API Key 等敏感信息已脱敏

        ❓ 这个 tab 是什么？怎么用？
        

          
调试 (Debug) — 展示本次分析加载的策略文件与经验库条目，便于排查 prompt 构造问题。

          
3 个卡片：

          
    
            
  • 📄 阶段一策略文件 — 静态注入阶段一 system prompt 的 .txt 文件清单
    • 
            
  • 📄 阶段二策略文件 — 阶段二动态加载的 .txt 文件清单（根据方向 / 形态路由）
    • 
            
  • 📚 经验库 — 加载的成功 / 失败案例数量 + 文件名列表
    • 
          

          
使用建议：当 AI 决策明显不符合策略时，检查本 tab 看是否漏加载了某个关键策略文件；想看完整 prompt 内容去【原始】tab
