# 获取数据（基础功能）

本文说明如何从数据源获取 K 线、刷新机制、切换逻辑与常见问题，覆盖两种运行模式：

- **Web 后端**（推荐）：FastAPI + SSE 实时推送，前端自动刷新
- **桌面 GUI**：PyQt6 + RefreshLoop 定时轮询，需手动点击「获取数据」

---

## 1. 支持的数据源

| 来源 | 说明 | 是否需登录 | 支持的品种 |
|---|---|---|---|
| **TradingView** | tvDatafeed 拉取，默认推荐 | 匿名可用，可选登录 | 加密货币、外汇、现货黄金、A 股、港股、美股、指数、期货 |
| **AkShare** | 东财数据，A股专用 | 无需 | A 股（沪深）股票与指数 |
| **MT5** | MetaTrader 5 终端 | 需 MT5 终端运行并登录 | 全部 MT5 时间周期（1m ~ 1M） |
| yfinance / eastmoney / tushare | 代码支持但 UI 未暴露 | - | - |

> ⚠️ Web 后端模式下，TradingView 数据源需要安装 tvDatafeed：`pip install git+https://github.com/rongardF/tvdatafeed.git`。若未安装，可切换到 AkShare（A 股数据源，无需 tvDatafeed）作为 fallback。

---

## 2. 操作步骤

### 2.1 Web 后端（推荐）

1. 启动服务后浏览器访问 http://localhost:8000
2. 在顶部工具栏选择 **数据源** / **交易所** / **品种** / **周期**
3. 切换后前端自动调用 `/api/subscribe`，SSE 流自动重连
4. 图表开始刷新，状态栏显示「SSE 实时 · 距上次刷新 Ns · 距下次收盘 Ms」
5. 状态栏倒计时归零时表示当前 K 线收盘，下一根开始形成

**不需要点击「获取数据」按钮**，切换即订阅，SSE 自动推送。

### 2.2 桌面 GUI（原项目主入口）

1. 在控制栏选择 **数据来源**（MT5 / TradingView）
2. 选择 **品种**（如 `XAUUSDm`、`600519`）、**周期**（如 15m、1h、1d）
3. 若数据来源为 TradingView，可选 **交易所**（OANDA / PEPPERSTONE / TVC 等；默认「自动」自动探测）
4. 点击 **「获取数据」** 按钮
5. 图表开始刷新，状态栏显示当前数据来源和品种信息；等待状态栏确认数据获取成功

---

## 3. 刷新机制

### 3.1 Web 后端：SSE 主动推送

- **端点**：`/api/bars/stream`（Server-Sent Events）
- **事件类型**：
  - `bar_update`：forming bar 期间增量推送（每 1s）
  - `bar_close`：当前 K 线收盘，下一根开始
  - `ping`：心跳保活（避免网关超时）
- **状态栏显示**：`● SSE 实时 · 距上次刷新 Ns · 距下次收盘 Ms`
- **降级模式**：SSE 连接失败时自动切换到 3s 轮询 + 5s 拉取 `next-close`，仍显示倒计时

### 3.2 桌面 GUI：定时轮询

- 点击「获取数据」后，程序以设定间隔（默认 **1 秒**）持续从数据源拉取最新 K 线
- 新数据到达后自动更新图表（蜡烛图 + EMA20 叠加线）
- 最新未收盘 K 线以**浅色空心蜡烛**显示，不参与 AI 分析
- **切换品种 / 周期后，刷新自动停止**，需重新点击「获取数据」才会恢复拉取

---

## 4. TradingView 交易所自动探测

选择交易所为「（自动）」时，程序按以下优先级自动探测可用数据源：

1. **A 股**：先尝试上交所（SSE），再尝试深交所（SZSE）
2. **港股**：使用港交所（HKEX）
3. **加密货币**：先尝试 BINANCE，再依次尝试 BITSTAMP、COINBASE 等
4. **指数**：SPX → NDX → VIX 等使用预设交易所
5. **黄金 / 外汇**：按 OANDA → PEPPERSTONE → FOREXCOM → TVC 顺序探测

探测过程中状态栏实时显示当前尝试的交易所；首个返回数据的交易所将被采用。

---

## 5. 切换品种/周期/交易所

### 5.1 Web 后端

- 切换通过 `/api/subscribe` API，前端使用 `Promise.allSettled` **并行执行** settings 更新、品种列表拉取、首根 K 线获取
- 新增 `_inflightSwitch` 防重入机制，避免快速连续点击导致请求混乱
- 品种列表缓存：`Map<exchange, symbolList>` + 10 分钟 TTL，避免重复拉取
- API 请求带 15 秒 `AbortController` 超时
- 切换成功后 SSE 自动重连，无需手动操作

### 5.2 桌面 GUI

- 切换数据来源时，图表自动重置，旧数据源断开连接
- 切换品种 / 周期时，图表自动重置，刷新停止，需重新点击「获取数据」
- 切换期间若有分析正在进行，分析将被取消

## 6. 连接失败处理

| 场景 | Web 后端行为 | 桌面 GUI 行为 |
|---|---|---|
| **MT5 未连接** | `/api/health/check` 返回 `error` | 状态栏提示「数据源未连接」 |
| **TradingView 不可达** | 健康检查 `degraded`，SSE 降级到轮询 | 弹出连接失败对话框 |
| **品种名无效** | `/api/subscribe` 返回 `error_type: symbol`，前端 `showSwitchError` 提示 | 品种输入框下方红色警告 |
| **网络波动** | SSE 自动重连，重连失败降级到 3s 轮询 | 刷新循环指数退避（0.5s → 1s → 2s → … 最大 10s） |
| **请求超时** | 后端 15s 超时返回 `error_type: timeout` | - |
| **tvDatafeed 未安装** | TradingView 切换时报 `No module named 'tvDatafeed'`，建议切换到 AkShare | 同 |

后端错误响应统一包含 `error_type: connection|symbol|timeout` 分类，前端按类型展示友好提示。响应头 `Cache-Control: max-age=600` 缓存品种列表。

## 7. 注意事项

- 品种名与 MT5「市场报价」中名称必须完全一致（含后缀如 `XAUUSDm`）
- K 线数据不足时，图表缓冲区需要数秒填满，期间状态栏提示「数据不足」
- TradingView 匿名模式数据可能受限，部分品种需登录账号（通过 `.env` 的 `PA_AGENT_TRADINGVIEW_USERNAME` / `PA_AGENT_TRADINGVIEW_PASSWORD` 配置）
- 刷新间隔过短可能触发 TradingView 限流，Web 后端 SSE 默认每 1s 推送一次
- TradingView 连通性在切换品种时按需检测，而非程序启动时检测
- **休市时**：数据源只返回已收盘 K 线，不返回 forming bar；`latest_snapshot(n)` 返回 `n` 根全 closed（详见 [图表K线与分析快照说明.md](./图表K线与分析快照说明.md) §5）

## 8. 关键 API（Web 后端）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/datasources` | 列出可选数据源 |
| GET | `/api/timeframes` | 当前数据源支持的周期 |
| GET | `/api/tv/symbols?exchange=&q=` | 搜索品种（返回代码+中文名称） |
| POST | `/api/subscribe` | 切换品种/周期/数据源（TradingView 可带 `exchange`） |
| GET | `/api/bars?count=N` | 拉取最新 N 根 K 线（图表用） |
| GET | `/api/bars/stream` | **SSE 实时 K 线流**（K 线收盘/新 K 线形成时推送） |
| GET | `/api/bars/next-close` | 查询下一 bar 收盘时间戳（降级轮询模式用） |
| GET | `/api/health/check` | 完整健康检查（模型 API + 数据源连通性） |
